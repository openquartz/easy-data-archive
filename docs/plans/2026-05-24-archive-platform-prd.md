# 归档平台产品需求文档（PRD）

## 1. 产品目标
- 建设一套统一的数据归档管理平台，覆盖权限、配置、执行、日志、告警、可视化全流程。
- 让管理员可通过页面完成数据源接入、归档分组配置、规则编排、任务触发与取消。
- 让审计与运维角色可查看日志、告警、执行明细和核心指标。

## 2. 角色定义
- 平台管理员：负责用户、角色、权限、数据源、监控规则、全局配置管理。
- 归档管理员：负责归档分组、规则、任务触发、任务取消、日志查看。
- 审计员：负责查看操作日志、任务日志、告警，不可修改配置。
- 观察员：仅查看任务状态、大盘、基础信息。

## 3. 权限模型
- 采用“菜单 + 按钮 + 数据范围”三层模型。
- 一个用户可绑定多个角色。
- 数据范围至少支持：全部、指定分组、指定数据源、本人负责。
- 高危权限必须独立授权：测试连接、手动触发、取消归档、删除规则、角色授权。

## 4. 模块需求

### 4.1 基础权限模块

#### 4.1.1 登录/登出
- 用户通过账号密码登录。
- 登录成功后签发 Token，并写入登录会话表。
- 登录失败需记录失败原因、IP、终端信息、失败时间。
- 登出时服务端失效当前会话，前端清理 Token 和权限缓存。
- 被禁用账号、角色失效账号、密码错误超限账号禁止登录。

#### 4.1.2 多维度账号分配与角色绑定
- 支持创建、编辑、禁用、启用用户。
- 支持用户绑定多个角色。
- 支持角色绑定多个权限码。
- 支持用户配置负责人属性、手机号、邮箱、告警接收方式。
- 支持数据范围授权，限制用户仅操作被授权的数据源或分组。

#### 4.1.3 全链路操作日志
- 必须采集以下行为：登录、登出、创建、编辑、删除、测试连接、触发归档、取消归档、角色授权、状态变更。
- 日志字段包括：操作人、模块、操作类型、请求路径、请求参数摘要、结果状态、响应码、耗时、IP、时间。
- 支持按用户、模块、操作类型、结果状态、时间范围查询。
- 支持查看详情和导出。

### 4.2 数据归档核心模块

#### 4.2.1 数据源管理
- 支持新增、编辑、启停、查看、测试连接。
- 关键字段包括：编码、名称、类型、JDBC URL、用户名、密码、Schema、备注、负责人。
- 新增或编辑后支持即时连接校验。
- 连接校验至少验证：数据库可达、账号可登录、目标 Schema 可访问、基础 SQL 可执行。
- 密码展示默认脱敏，编辑时支持“保持原值”。

#### 4.2.2 归档分组层级管理
- 分组支持多级树形结构。
- 分组绑定源数据源、目标数据源、负责人、描述、启用状态。
- 同级分组编码唯一。
- 禁用分组后不可创建新任务。
- 删除分组前必须确保无启用规则、无运行中任务。

#### 4.2.3 归档规则管理
- 每个分组下支持多条规则。
- 规则类型至少支持：按时间归档、按 ID 归档。
- 支持多条件匹配：字段、运算符、值、逻辑关系、顺序。
- 支持优先级，数值越小优先级越高。
- 支持配置来源表、目标表、ID 字段、抓取 SQL、删除保护条件、批大小、滚动窗口、停顿时间、是否写入目标、是否删除源数据。
- 保存时必须做完整校验：字段完整性、优先级冲突、条件合法性、SQL 模板可编译、表达式可解析。

#### 4.2.4 归档进度与速率监控
- 手动触发后需生成任务并实时更新状态。
- 状态流转至少支持：`WAITING -> RUNNING -> SUCCESS/FAILED/CANCELLING/CANCELLED`。
- 进度上报字段至少包括：已处理条数、当前规则、当前阶段、处理速率、心跳时间、异常摘要。
- 前端默认轮询实时进度。
- 当心跳超时未更新时，平台需判定异常并触发告警。

#### 4.2.5 执行日志存储与检索
- 日志级别支持 `INFO/WARN/ERROR`。
- 日志类型至少包括：任务开始、规则开始、批次进度、取消申请、任务完成、任务失败。
- 支持按任务、分组、规则、日志级别、执行阶段、时间范围查询。
- 日志详情需支持查看 SQL 摘要、批次数、处理条数、异常信息摘要。

#### 4.2.6 手动触发与中途取消
- 手动触发前必须校验：分组启用、规则有效、源目标数据源可用、同分组无运行中任务。
- 取消任务时仅允许 `WAITING`、`RUNNING`、`CANCELLING` 状态发起。
- 取消成功受理后，前端立即显示 `CANCELLING`。
- 执行线程在下一个批次边界感知取消标记，完成状态同步为 `CANCELLED`。
- 取消需记录取消人、取消时间、取消原因。

### 4.3 数据可视化模块

#### 4.3.1 大盘汇总指标
- 今日任务总数、成功数、失败数、取消数。
- 今日归档总条数、平均速率、峰值速率。
- 近 7 天执行趋势，支持按分组和数据源维度统计。
- Top 风险分组：失败最多、耗时最长、速率最低。
- 当前未处理告警数、最近异常任务、最近失败数据源。

#### 4.3.2 监控预警
- 支持阈值配置：任务失败、心跳超时、速率低于阈值、错误日志数超限、连接测试失败。
- 支持推送渠道：邮件、Webhook，企业微信/钉钉预留扩展。
- 告警事件需记录等级、触发规则、关联任务、通知状态、恢复状态、处理人。
- 支持静默时间、重复告警收敛、告警恢复。

## 5. 交互逻辑

### 5.1 登录
- 登录成功后进入首页，立即加载用户信息、角色权限、动态菜单。
- 登录失败时页面显示明确错误提示，同时后端写失败日志。
- Token 失效时前端统一跳转登录页。

### 5.2 规则配置
- 新建规则时先选规则类型，再渲染差异化表单。
- 条件编辑器支持增删、排序、实时预览。
- 点击“保存并校验”后返回合法性结果。
- 校验不通过时禁止启用。

### 5.3 任务中心
- 列表默认按开始时间倒序。
- 任务详情页展示规则明细、进度、速率、执行日志。
- 运行中任务显示取消按钮、当前规则、最近心跳。
- 无权限用户不可见高危按钮。

## 6. 业务规则汇总
- 同一分组内规则优先级必须唯一。
- 同一分组同一时刻仅允许一个运行中任务。
- 启用规则前必须校验分组启用且关联数据源可用。
- 写入关闭且源清理开启的规则默认不允许启用。
- 禁用数据源后，引用该数据源的分组不可触发新任务。
- 已完成任务不可取消，但可重新触发同分组新任务。

## 7. 权限管控要求
- `auth:login`：匿名开放。
- `auth:logout`：已登录用户。
- `user:*`、`role:*`、`permission:*`：平台管理员。
- `datasource:test`、`datasource:edit`：平台管理员、归档管理员。
- `archive:group:*`、`archive:rule:*`：具备归档管理权限且受数据范围限制。
- `archive:task:trigger`、`archive:task:cancel`：高危权限，建议前端二次确认。
- `log:operation:view`、`log:task:view`：审计员及以上。
- `dashboard:view`：观察员及以上。

## 8. 验收标准
- 用户可完成登录/登出并正确加载权限。
- 数据源新增、编辑、测试连接流程完整可用。
- 分组和规则可正常配置并通过校验。
- 手动触发任务后可实时查看进度、速率和日志。
- 运行中任务可成功取消，前后端状态一致。
- 大盘与监控页可正确展示指标与告警。
- 所有高危操作均可在操作日志中追溯。
